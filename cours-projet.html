1 Introduction

daems-site est un site fan non officiel pour le streamer Twitch daems_. Il permet à la communauté de se connecter avec leur compte Twitch, de voir le stream en direct, de parcourir les clips récents, de chatter en temps réel, et de voir leur bannière personnalisée selon leur durée d'abonnement.

Ce projet est un bon exemple d'une SPA (Single Page Application) moderne qui combine une API tierce (Twitch), des WebSockets, du serverless, et une base de données clé-valeur dans le cloud.

Objectif pédagogique : Comprendre comment construire une app web complète en partant de zéro — de l'authentification OAuth au déploiement serverless, en passant par la gestion d'état React et les protocoles réseau.

2 Stack technique

4 OAuth Twitch — Connexion

Twitch utilise le protocole OAuth 2.0. On utilise ici le flux Implicit Grant : le token d'accès est retourné directement dans l'URL (fragment #), sans passer par un serveur backend pour l'échange de code.

1. Clic "Se connecter"
→
Redirection vers id.twitch.tv/oauth2/authorize
2. Utilisateur accepte les scopes
→
Twitch redirige vers /?#access_token=xxx
3. React lit le hash URL
→
Token stocké dans localStorage

Code de connexion

// Scopes demandés à Twitch
const SCOPES = [
  "user:read:email",
  "user:read:follows",
  "user:read:subscriptions",
  "chat:read",
  "chat:edit"
].join(" ");

// Redirection vers Twitch
const login = () => {
  window.location.href = `https://id.twitch.tv/oauth2/authorize
    ?response_type=token
    &client_id=${CLIENT_ID}
    &redirect_uri=${encodeURIComponent(REDIRECT_URI)}
    &scope=${encodeURIComponent(SCOPES)}`;
};

// Lecture du token dans le hash URL au chargement
useEffect(() => {
  const hash = window.location.hash;
  if (hash.includes("access_token=")) {
    const p = new URLSearchParams(hash.slice(1));
    const t = p.get("access_token");
    if (t) {
      localStorage.setItem("tw_token", t);
      setToken(t);
      window.history.replaceState(null, "", window.location.pathname);
    }
  }
}, []);

Ce qu'on apprend

  • Comment fonctionne OAuth 2.0 Implicit Grant
  • Lire des paramètres dans le hash de l'URL (window.location.hash)
  • Persister un token dans localStorage pour ne pas se reconnecter à chaque visite
  • Nettoyer l'URL avec history.replaceState après extraction du token

5 Chat IRC — WebSocket

Twitch Chat utilise le protocole IRC (Internet Relay Chat) exposé via WebSocket. Toute la logique est dans le hook useIRC, qui gère la connexion, le parsing des messages, les badges et les notifications système.

Cycle de vie du WebSocket

const ws = new WebSocket("wss://irc-ws.chat.twitch.tv");

ws.onopen = () => {
  // Demande les tags enrichis (badges, couleurs, timestamps…)
  ws.send("CAP REQ :twitch.tv/tags twitch.tv/commands");
  ws.send(`PASS oauth:${token}`);
  ws.send(`NICK ${username}`);
  ws.send(`JOIN #daems_`);  // rejoindre le channel
};

ws.onmessage = (e) => {
  // Répondre aux PING pour maintenir la connexion
  if (raw.startsWith("PING")) ws.send("PONG :tmi.twitch.tv");

  // Parser les messages PRIVMSG (messages du chat)
  if (raw.includes("PRIVMSG")) { /* ... extraction des données ... */ }

  // USERNOTICE = événements (sub, raid, etc.)
  if (raw.includes("USERNOTICE")) { /* ... message système ... */ }
};

Parsing des tags IRC

Twitch envoie des métadonnées riches dans les tags IRC, préfixés par @ :

// Format brut d'un message IRC Twitch :
@badge-info=subscriber/14;badges=subscriber/12;color=#FF4500;
 display-name=SamUser;id=abc-123;mod=0;
 PRIVMSG #daems_ :Gg ez !

// Fonction de parsing :
const parseTags = (str) => {
  const tags = {};
  str.split(";").forEach(t => {
    const [k, ...rest] = t.split("=");
    tags[k] = rest.join("=") || "";
  });
  return tags;
};

Messages locaux (optimistic UI)

Quand l'utilisateur envoie un message, il ne reçoit pas son propre message via IRC immédiatement. La solution : ajouter le message localement tout de suite, puis l'IRC le confirme. 

const send = () => {
  const sent = sendIRC(input.trim());
  if (sent) {
    // Ajout local immédiat (pas besoin d'attendre l'IRC)
    setLocalMessages(prev => [...prev, {
      id: `local-${Date.now()}`,
      displayName: userInfo.display_name,
      text: input.trim(),
      isLocal: true,
      ts: Date.now(),
    }]);
  }
};

// Fusion IRC + local, déduplication par texte
const ircTexts = new Set(ircMessages.map(m => m.text));
const filteredLocal = localMessages.filter(m => !ircTexts.has(m.text));
Optimistic UI : On affiche le résultat attendu avant la confirmation serveur pour donner une sensation de réactivité immédiate. Technique courante dans les apps de messagerie (Slack, WhatsApp…).

6 Bannières de rang

Chaque viewer connecté reçoit une bannière personnalisée selon son nombre de mois d'abonnement, récupéré via le tag IRC badge-info=subscriber/N.

Tier Mois Rang Étoiles Spécial
0 – Bronze ⭐ –
1 0–2 mois Silver ⭐⭐ –
2 3–5 mois Gold ⭐⭐⭐ –
3 6–11 mois Platinum ⭐⭐⭐⭐ –
4 12+ mois Diamond ⭐⭐⭐⭐⭐ ✨ Shimmer animé

Logique de calcul du tier

function getBannerTier(subMonths, isSub) {
  if (!isSub || subMonths === null) return 0;  // Bronze (pas sub)
  if (subMonths < 3)  return 1;               // Silver
  if (subMonths < 6)  return 2;               // Gold
  if (subMonths < 12) return 3;               // Platinum
  return 4;                                    // Diamond
}

8 Styling — CSS inline

Tout le style est écrit en objets JavaScript passés à la prop style. Pas de classes CSS, pas de framework comme Tailwind ou Bootstrap.

Avantages / inconvénients

✅ Avantages

Styles colocalisés avec le composant. Pas de conflit de classes CSS. Valeurs dynamiques faciles (couleur selon le tier, etc.).

⚠️ Inconvénients

Impossible d'utiliser des pseudo-classes (:hover) directement. Verbose pour les gros composants.

Contournement pour le hover

// Solution : événements onMouseEnter / onMouseLeave
<div
  onMouseEnter={e => { e.currentTarget.style.transform = "translateY(-4px)"; }}
  onMouseLeave={e => { e.currentTarget.style.transform = ""; }}
>

Animations CSS globales

Les keyframes CSS (impossibles en inline) sont injectées via une balise <style> dans le JSX :

const CSS = `
  @keyframes shimmer {
    0%   { transform: translateX(-100%); }
    100% { transform: translateX(200%); }
  }
  @keyframes pulse { ... }
  @keyframes spin  { to { transform: rotate(360deg); } }
  @keyframes ticker {
    0%   { transform: translateX(0); }
    100% { transform: translateX(-50%); }
  }
`;

// Injection dans le DOM
return (
  <div>
    <style>{CSS}</style>

  </div>

9 Routes API Vercel

Chaque fichier dans api/ devient automatiquement une fonction serverless sur Vercel accessible à l'URL /api/nom-du-fichier.

Pourquoi un backend ? Certains appels Twitch nécessitent un token de broadcaster (accès aux listes de subs). Ce token NE peut PAS être exposé côté client — il doit rester dans les variables d'environnement Vercel.
POST /api/store-viewer Sauvegarde viewer + sub_months dans KV
GET /api/recent-activity Derniers subs + top bits de la semaine
GET /api/subscribers Liste triée (tier, ancienneté follow)
GET /api/sub-count Nombre total d'abonnés
GET /api/followers Nombre de followers
GET /api/bits Leaderboard bits
POST /api/save-token Admin : sauvegarde broadcaster token (protégé par ADMIN_SECRET)

Structure d'un handler Vercel

export default async function handler(req, res) {
  // 1. Headers CORS (permettre les appels depuis le frontend)
  res.setHeader("Access-Control-Allow-Origin", "*");

  // 2. Token broadcaster depuis les env vars (jamais dans le code !)
  const token = process.env.BROADCASTER_TOKEN;
  if (!token) return res.status(500).json({ error: "Token manquant" });

  // 3. Appel Twitch Helix API
  const r = await fetch("https://api.twitch.tv/helix/...", {
    headers: { "Authorization": `Bearer ${token}`, "Client-Id": CLIENT_ID }
  });

  // 4. Retourner la réponse JSON
  return res.status(200).json(await r.json());
}

10 Upstash KV (Redis)

Upstash est une base Redis serverless (sans connexions persistantes). On l'utilise via @vercel/kv pour stocker les données des viewers et construire des leaderboards.

Structures de données utilisées

Hash — viewer:{user_id}

Stocke les infos d'un viewer : display_name, sub_months, is_sub, updated_at.

Sorted Set — leaderboard:sub_months

Classement par mois d'abonnement. Score = sub_months. Permet ZRANGE en ordre décroissant.

import { kv } from "@vercel/kv";

// Sauvegarder les données d'un viewer
await kv.hset(`viewer:${user_id}`, {
  display_name,
  sub_months: sub_months || 0,
  is_sub: is_sub || false,
  updated_at: Date.now(),
});

// Ajouter au leaderboard si sub (score = nb de mois)
if (is_sub && sub_months > 0) {
  await kv.zadd("leaderboard:sub_months", {
    score: sub_months,
    member: user_id
  });
}

11 Custom Hooks React

Un custom hook est une fonction préfixée use qui encapsule de la logique réutilisable avec des hooks React (useState, useEffect, useRef…).

useIRC — Anatomie complète

function useIRC(token, username) {
  // États internes
  const [subMonths, setSubMonths] = useState(null);
  const [ircMessages, setIrcMessages] = useState([]);
  const [connected, setConnected] = useState(false);
  const wsRef = useRef(null);  // ref stable, pas de re-render

  // Connexion WebSocket (re-run si token ou username change)
  useEffect(() => {
    if (!token || !username) return;
    const ws = new WebSocket("wss://irc-ws.chat.twitch.tv");
    wsRef.current = ws;
    /* ... handlers ... */
    // Cleanup : ferme le WebSocket si le composant est démonté
    return () => { ws.close(); };
  }, [token, username]);

  // Fonction stable (useCallback évite les re-renders inutiles)
  const sendIRC = useCallback((msg) => {
    const ws = wsRef.current;
    if (!ws || ws.readyState !== WebSocket.OPEN) return false;
    ws.send(`PRIVMSG #daems_ :${msg}`);
    return true;
  }, []);

  return { subMonths, ircMessages, connected, sendIRC, parseBadges };
}

Concepts React utilisés

useRef

Référence mutable qui ne déclenche pas de re-render. Utilisé pour stocker l'instance WebSocket et les refs DOM (bottomRef, inputRef).

useCallback

Mémoïse une fonction pour éviter de la recréer à chaque render. Essentiel quand la fonction est passée en prop ou dans un useEffect.

useEffect cleanup

La fonction retournée par useEffect s'exécute au démontage du composant. Ici, elle ferme le WebSocket proprement.

useState avec initializer

useState(() => localStorage.getItem("tw_token")) — passer une fonction évite de lire le localStorage à chaque render.

12 Patterns UI récurrents

Couleur déterministe par pseudo-random

Assigner une couleur fixe à chaque username de manière consistante, sans la stocker :

const COLORS = ["#FF4500", "#FF69B4", "#9ACD32", "#00FF7F", ...];

const colorFor = (name) =>
  COLORS[name.split("").reduce((a, c) => a + c.charCodeAt(0), 0) % COLORS.length];
// Même input → même couleur, toujours

Twitch renvoie un cursor dans la réponse pour charger la page suivante :

const data = await twitchGet("/clips", token, params);
const nextCursor = data.pagination?.cursor || null;
// Pour la page suivante : params.after = nextCursor

const bottomRef = useRef(null);

13 Déploiement & Variables d'environnement

Le projet est déployé sur Vercel. Le build Vite génère les fichiers statiques, et les fonctions api/*.js deviennent des Lambda functions.

Variables d'environnement

Sécurité : Ne jamais committer ces valeurs dans le code. Elles sont dans .env.local en dev (non-commité) et dans le dashboard Vercel en prod.
BROADCASTER_TOKEN

Token OAuth du broadcaster. Donne accès aux endpoints Twitch privés (liste des subs, etc.).

KV_REST_API_URL + KV_REST_API_TOKEN

Credentials Upstash pour la base de données Redis.

ADMIN_SECRET

Secret pour protéger la route /api/save-token (accessible seulement par l'admin).

CLIENT_ID (hardcodé)

L'ID de l'app Twitch est dans le code car il est public par nature (comme une API key publique).
